# Contributing to Vivify QuestP0RT

Thank you for your interest in contributing! Here are some guidelines:

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch (`git checkout -b feature/AmazingFeature`)
4. Make your changes
5. Commit your changes (`git commit -m 'Add AmazingFeature'`)
6. Push to the branch (`git push origin feature/AmazingFeature`)
7. Open a Pull Request

## Development Setup

- Install Beat Saber modding tools (BeatSaber-IPA, QuestAPI)
- Set up your IDE with C++ support (Visual Studio, VS Code, etc.)
- Follow the existing code style and patterns
- Ensure you have the Beat Saber SDK installed

## Testing

Please test your changes thoroughly:
- Test with custom notes
- Test with custom sabers
- Verify no crashes occur
- Ensure compatibility with original game features
- Test on actual PCVR hardware when possible

## Code Standards

- Follow existing code style and conventions
- Add comments for complex logic
- Test before submitting PR
- Update documentation as needed
- Keep commits atomic and well-described

## Reporting Bugs

- Use GitHub Issues
- Include Beat Saber version
- Describe reproduction steps clearly
- Include relevant logs and error messages
- Specify your PCVR hardware setup

## Feature Requests

- Check if feature already exists or is in progress
- Explain use case and benefits
- Be open to discussion and feedback
- Consider backwards compatibility

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and improve
- Report inappropriate behavior

Thank you for contributing to Vivify QuestP0RT!
